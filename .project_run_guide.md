# Portfolio Project Suite - Execution & Verification Guide

This guide provides step-by-step instructions and a verification checklist for manually running and verifying all 5 projects in the automation testing portfolio.

## Project 1: E-commerce Testing (`project-1-ecommerce-testing`)
**Objective**: Verify Playwright E2E UI testing on an e-commerce platform using the Page Object Model (POM).

### Execution Steps:
1. Open terminal and navigate: `cd project-1-ecommerce-testing`
2. Install dependencies: `npm install`
3. Install Playwright browsers (if needed): `npx playwright install`
4. Run all tests: `npm run test`
5. (Optional) Run visually: `npm run test:ui`
6. View the HTML report: `npm run report`

### Verification Checklist:
- [ ] Dependencies installed without major errors.
- [ ] Playwright tests passed across browsers (Chromium, Firefox, WebKit).
- [ ] HTML report generated and correctly displays inside the browser.
- [ ] Screenshots, videos, or traces are captured in `test-results/` for any failed tests.

---

## Project 2: API Testing (`project-2-api-testing`)
**Objective**: Verify Playwright REST API tests and Python unittest contract tests.

### Execution Steps:
1. Navigate to project: `cd project-2-api-testing`
2. Map environment variables: `cp .env.example .env` (Populate keys if needed)
3. Install Node.js dependencies: `npm install`
4. Run Playwright API test suite: `npm run test`
5. Run Python unittests (if you have Python configured): `npm run python:test`
6. View the final HTML report: `npm run report`

### Verification Checklist:
- [ ] `.env` file successfully created from `.env.example`.
- [ ] Playwright tests for multiple endpoints (Users, Products, Orders) executed and passed.
- [ ] Python unittest suite ran and passed the data assertions.
- [ ] API JSON schemas validated correctly in the test context.

## Project 3: CI/CD Pipeline (`project-3-cicd-pipeline`)
**Objective**: Verify Jenkins and Docker orchestration for the automated tests.

### Execution Steps:
1. Navigate to project: `cd project-3-cicd-pipeline`
2. Start the Docker environment: `docker-compose up -d`
3. Verify the Jenkins container is running on `http://localhost:8080` (or similar configured port).
4. Review the `Jenkinsfile` for valid staging structures.
5. Tear down the environment when done: `docker-compose down`

### Verification Checklist:
- [ ] `docker-compose up` pulled and built images without syntax issue.
- [ ] Jenkins dashboard is accessible.
- [ ] Docker containers shut down cleanly when running `docker-compose down`.

## Project 4: Advanced Framework (`project-4-advanced-framework`)
**Objective**: Verify enterprise software patterns (Singleton, Factory, Strategy) incorporated with Playwright & Jest.

### Execution Steps:
1. Navigate to project: `cd project-4-advanced-framework`
2. Install dependencies: `npm install`
3. Run design-pattern Unit tests (Jest): `npm run test:unit`
4. Run E2E test suite (Playwright): `npm run test:e2e`
5. View report: `npm run report`

### Verification Checklist:
- [ ] `npm install` resolves dependencies.
- [ ] Jest safely executed and tested the architecture patterns without error.
- [ ] Playwright E2E tests passed using the framework patterns.
- [ ] Logs generated in the `logs/` directory using the Winston logger.

---

## Project 5: Full Stack Platform (`project-5-full-stack-platform`)
**Objective**: Verify the complete Orchestrator for E2E, API, and K6 Performance testing.

### Execution Steps:
1. Navigate to project: `cd project-5-full-stack-platform`
2. Install dependencies: `npm install`
3. Setup the docker infrastructure: `npm run docker:up`
4. Run the core orchestrator script: `npm run start` (Runs TestOrchestrator.ts)
5. Run Playwright E2E tests manually: `npm run test:e2e`
6. Run K6 Performance tests (requires K6 installed locally): `npm run test:k6`
7. View final Playwright results: `npm run report`
8. Teardown: `npm run docker:down`

### Verification Checklist:
- [ ] `docker-compose` spun up monitoring/services correctly.
- [ ] `npm run start` executed the TypeScript orchestrator cleanly.
- [ ] K6 load tests (`load-test.js`) ran and spat out load performance metrics to stdout.
- [ ] Playwright E2E suites passed.
